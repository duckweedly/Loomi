---
title: 开发同步文档规则
description: 每次非平凡开发都必须同步更新 Loomi 文档站。
---

Loomi 的默认开发规则是：代码变化和技术文档同步发生。除非只是修拼写、格式化、删除无用空白或无行为变化的小改动，否则开发者和 AI Agent 必须检查并更新文档站。

## 必须更新的内容

如果变更影响功能目标、用户流程或验收标准，应更新对应 Spec Kit 规格说明，并在文档站的 Spec Kit 或路线页面中记录当前状态。

如果变更影响模块边界、数据流、状态机、事件模型、权限、安全或运行方式，应更新 `architecture/` 下的相关页面。

如果变更引入重要技术取舍，例如新增依赖、选择数据库 schema、改变运行模式、改变协议或改变测试策略，应新增或更新 `adr/` 页面。

如果变更完成了一个阶段性任务，应在 `devlog/` 增加开发日志，记录本轮做了什么、如何验证、遗留什么、下一步是什么。

如果变更影响命令、环境变量、启动方式、测试方式或排错方式，应更新对应 quickstart/runbook 页面。

## 开发日志格式

每条开发日志至少包含：日期、关联功能或阶段、完成内容、验证结果、已知限制和下一步。

验证结果必须真实记录。构建或测试失败时，不能写成通过；应记录失败命令、关键错误和处理计划。

## AI Agent 执行要求

AI Agent 在 Loomi 中执行非平凡开发时，不需要用户额外提醒，必须主动同步更新文档站。实现完成前应检查：代码是否已验证、Spec Kit 产物是否需要更新、`docs-site/` 是否需要新增或修改页面、文档站本身是否能构建通过。

如果文档更新会引入明显重复或还缺少产品决策，Agent 应在最终回复中说明原因；能自己从代码、计划和规格中确定的事实，不应等待用户提醒。
